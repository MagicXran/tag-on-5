# OA同步拆分数据说明

## 功能确认

经费号拆分功能已经与OA同步完全集成。拆分后的每条记录都会作为独立的记录同步到OA系统。

## 数据流程

### 1. Excel读取和拆分
```
原始数据：
合同编号: HT001, 经费号: "3932001,3932002,3932003"
↓
拆分后：
合同编号: HT001, 经费号: "3932001"
合同编号: HT001, 经费号: "3932002"  
合同编号: HT001, 经费号: "3932003"
```

### 2. 数据库保存
- 使用复合主键 (contractid + fundids) 保存每条拆分后的记录
- 每条记录都有独立的数据库ID

### 3. OA同步
- 每条拆分后的记录都会作为独立的OA表单记录
- OA系统中会看到3条独立的合同记录，每条只有一个经费号

## 示例

假设有一个合同包含3个经费号：

**Excel原始数据：**
```
合同编号: HT2025001
合同名称: 技术开发合同
经费号: 3932001,3932002,3932003
负责人: 张三
合同经费: 100万
```

**OA同步结果：**

OA系统中将创建3条独立的记录：

1. **记录1**
   - 合同编号: HT2025001
   - 合同名称: 技术开发合同
   - 经费号: 3932001
   - 负责人: 张三
   - 合同经费: 100万

2. **记录2**
   - 合同编号: HT2025001
   - 合同名称: 技术开发合同
   - 经费号: 3932002
   - 负责人: 张三
   - 合同经费: 100万

3. **记录3**
   - 合同编号: HT2025001
   - 合同名称: 技术开发合同
   - 经费号: 3932003
   - 负责人: 张三
   - 合同经费: 100万

## 技术实现

### 关键代码位置

1. **数据拆分** (`data_cleaner.py`):
   - `split_fundids_rows()` 方法负责拆分
   - `process_contracts_excel()` 调用拆分方法

2. **数据库保存** (`database_manager_sqlite.py`):
   - 复合主键: `UNIQUE(contractid, fundids)`
   - `save_contracts_data()` 保存拆分后的每条记录

3. **OA同步** (`main_processor.py` & `oa_sync_manager.py`):
   - `sync_contracts_to_oa()` 接收拆分后的记录列表
   - 每条记录独立同步到OA

### 数据结构

保存到数据库并传递给OA的记录格式：
```python
{
    "operation": "insert",
    "contractid": "HT2025001",
    "data": {
        "contractid": "HT2025001",
        "description": "技术开发合同",
        "fundids": "3932001",  # 单个经费号
        "leader": "张三",
        "contractfunds": 100.0,
        # ... 其他字段
    }
}
```

## 验证方法

1. **查看数据库**：
   ```sql
   SELECT contractid, fundids, description 
   FROM contracts 
   WHERE contractid = 'HT2025001'
   ORDER BY fundids;
   ```

2. **查看OA系统**：
   - 登录OA系统
   - 查看"科研合同档案"表单
   - 搜索合同编号，应看到多条记录

3. **运行测试脚本**：
   ```bash
   python test_oa_sync_split.py
   ```

## 注意事项

1. **数据一致性**：拆分后的每条记录除经费号外，其他字段完全相同
2. **OA权限**：确保OA账号有创建多条相同合同编号记录的权限
3. **更新处理**：更新时会根据contractid+fundids的组合进行匹配
4. **统计分析**：在进行数据统计时，注意同一合同可能有多条记录

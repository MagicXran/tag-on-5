# 经费号拆分功能说明

## 功能概述

该功能用于将合同清单中包含多个经费号（用逗号分隔）的记录拆分成多行，每行只包含一个经费号，其他字段保持不变。

## 实现原理

### 1. 数据处理流程

1. **读取Excel数据**：从Excel文件读取合同数据
2. **数据清洗**：进行常规的数据类型转换和清洗
3. **经费号拆分**：识别包含多个经费号的记录，拆分成多行
4. **数据保存**：将拆分后的数据保存到数据库

### 2. 拆分逻辑

在 `data_cleaner.py` 中新增了 `split_fundids_rows` 方法：

示例：
- 原始数据：合同HT001，经费号："3932001,3932002,3932003"
- 拆分结果：
  - 合同HT001，经费号："3932001"
  - 合同HT001，经费号："3932002"
  - 合同HT001，经费号："3932003"

### 3. 数据库结构调整

修改了 `contracts` 表的主键结构：
- **原主键**：contractid（合同编号）
- **新主键**：contractid + fundids（合同编号 + 经费号的复合主键）

这样允许同一个合同编号对应多个经费号的记录存在。

## 使用方法

该功能已集成到现有的合同数据处理流程中，无需额外操作：

1. **通过GUI界面**：正常使用 `python run_gui.py` 启动界面，选择合同文件夹进行处理
2. **通过命令行**：使用 `python run_processor.py` 处理合同数据

系统会自动识别并拆分包含多个经费号的合同记录。

## 注意事项

1. **数据完整性**：拆分后的每条记录除了经费号不同外，其他字段完全相同
2. **数据库更新**：如果之前已经导入过数据，需要注意数据库表结构的变化
3. **OA同步**：拆分后的每条记录都会作为独立的记录同步到OA系统
4. **历史数据**：如果需要处理历史数据，建议先备份数据库

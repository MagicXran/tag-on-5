# 经费到账单据号（unid）设计说明

## 变更日期
2025-11-27

## 变更背景

### 原有问题
经费到账清单原使用 `(fundid, allocation_date, contractid)` 作为联合主键判断记录唯一性。但实际业务中，同一个项目在同一天可能有多笔到账记录，导致：
1. 合法的多笔到账被误判为"重复记录"
2. 系统将新增记录错误地标记为"更新"
3. OA同步时出现数据不一致

### 业务需求
- 经费到账清单永远都是新增，没有更新操作
- 需要一个系统生成的唯一标识来区分每条记录
- 该标识需要同时保存到本地数据库和OA系统

## 设计方案

### 新增字段

| 字段名 | 数据库列名 | OA字段 | 类型 | 说明 |
|--------|------------|--------|------|------|
| 单据号 | unid | field0024 | VARCHAR(100) | 系统自动生成的UUID |

### 主键变更

| 变更前 | 变更后 |
|--------|--------|
| `(fundid, allocation_date, contractid)` 联合主键 | `unid` 单一主键 |

### 数据流程

```
Excel读取 → 数据清洗 → 筛选非空字段 → 生成UUID → 保存数据库 → 同步OA
                                        ↑
                                  新增步骤
```

## 代码修改

### 1. config.py

```python
# 主键配置
DATABASE_PRIMARY_KEYS = {
    "projectfunds": ["unid"],  # 改为单一主键
}

# 列名映射
PROJECT_FUNDS_COLUMN_MAPPING = {
    ...
    "单据号": "unid"  # 新增
}
```

### 2. database_manager_sqlite.py

- 表结构添加 `unid VARCHAR(100) NOT NULL UNIQUE` 字段
- `save_project_funds_data()` 方法改为纯插入逻辑，不再检查记录是否存在
- 现有记录自动生成UUID填充

### 3. data_cleaner.py

- 新增 `_generate_unid_for_project_funds()` 方法
- 在 `process_project_funds_excel()` 筛选后自动生成UUID

### 4. oa_sync_manager.py

- 字段映射添加 `"unid": "field0024"`
- 更新OA ID时使用unid作为主键

## 筛选逻辑（保持不变）

仍然筛选以下字段非空的记录：
- `fundid`（经费卡号）
- `allocation_date`（拨款时间）
- `contractid`（项目编号）

## 注意事项

1. **向后兼容**：现有数据库记录会自动生成unid填充
2. **重复导入**：同一Excel文件多次导入会产生多条记录（每次都是新增）
3. **OA端配置**：需确保OA系统的field0024字段已配置

## 数据库迁移

系统启动时自动执行：
1. 检查projectfunds表是否有unid字段
2. 如果没有，添加unid字段
3. 为现有记录生成UUID填充

## 接口兼容性

`save_project_funds_data()` 返回值保持不变：
```python
Tuple[List[Dict], List[Dict]]  # (inserted_records, updated_records)
```
- `inserted_records`: 所有新增的记录
- `updated_records`: 始终为空列表（经费数据不更新）

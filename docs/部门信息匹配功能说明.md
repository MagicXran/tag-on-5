# 部门信息匹配功能说明

## 功能概述

该功能为合同清单数据自动匹配负责人的部门信息。通过读取人员名单文件，根据合同中的"负责人职工号"字段，自动匹配并填充对应的部门信息。

## 实现原理

### 1. 数据流程

1. **人员名单文件检查**：在处理合同文件夹时，首先检查是否存在包含"人员名单"字样的Excel文件
2. **人员信息读取**：读取人员名单文件的前三列（A列：人员编号，B列：姓名，C列：部门）
3. **部门匹配**：处理合同数据时，根据"负责人职工号"与人员编号进行匹配
4. **数据保存**：将匹配到的部门信息保存到数据库的department字段
5. **OA同步**：同步到OA系统时包含部门信息（field0050）

### 2. 文件要求

#### 人员名单文件
- **文件名**：必须包含"人员名单"字样（如：学校人员名单.xlsx）
- **格式**：Excel文件（.xlsx 或 .xls）
- **内容要求**：
  - A列：人员编号（必填）
  - B列：姓名
  - C列：部门（必填）

示例：
| 人员编号 | 姓名 | 部门 |
|---------|------|------|
| EMP001  | 张三 | 工程技术研究院 |
| EMP002  | 李四 | 冶金工程研究院 |

#### 合同清单文件
- 必须包含"负责人职工号"字段
- 该字段的值应与人员名单中的"人员编号"对应

### 3. 使用方法

#### GUI界面操作
1. 启动GUI界面：`python run_gui.py`
2. 选择合同文件夹时，系统会自动检查人员名单文件
3. 如果没有找到人员名单文件，会显示错误提示：
   ```
   文件夹中未找到人员名单文件，请将人员名单文件放到文件夹 [文件夹路径] 中
   ```
4. 将人员名单文件放入合同文件夹后，重新选择文件夹并处理

#### 命令行操作
```bash
# 确保合同文件夹中包含人员名单文件
python run_processor.py
```

## 注意事项

1. **必需性**：人员名单文件是必需的，如果不存在会中止处理
2. **文件位置**：人员名单文件必须放在合同文件夹中
3. **匹配规则**：
   - 根据"负责人职工号"精确匹配
   - 如果找不到匹配的人员编号，部门字段为空
   - 如果合同没有"负责人职工号"或值为空，部门字段为空
4. **数据完整性**：部门信息会随合同数据一起拆分（如果合同有多个经费号）

## 常见问题

### Q1: 为什么提示找不到人员名单文件？
A: 请确保：
- 文件名包含"人员名单"字样
- 文件格式为Excel（.xlsx或.xls）
- 文件放在合同文件夹中，不是子文件夹

### Q2: 部门信息没有匹配上怎么办？
A: 检查：
- 合同清单中的"负责人职工号"是否与人员名单中的"人员编号"完全一致
- 人员名单中是否包含该职工号
- 职工号格式是否一致（如前后空格、大小写等）

### Q3: 如何更新人员名单？
A: 直接替换合同文件夹中的人员名单文件，重新运行处理即可

### Q4: OA系统中看不到部门信息？
A: 确认OA系统的表单配置中field0050字段是否正确配置为"部门"字段
